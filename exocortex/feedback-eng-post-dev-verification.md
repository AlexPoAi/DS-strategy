После ЛЮБОЙ инженерной/ремонтной работы (правка скриптов, фиксы scheduler/strategist/extractor) — следующий шаг всегда диагностика итога.

**Why:** Протокол ремонта из DS-strategy/CLAUDE.md секция "Инженерные работы": Диагностика → План → Риски → Исправление → **Диагностика итога**. Без post-check фикс может сломать что-то смежное.

**How to apply:** Сразу после коммита изменений в FMT-exocortex-template — запустить:
1. `bash ~/Github/FMT-exocortex-template/setup/validate-template.sh`
2. Симуляция изменённого сценария вручную
3. Проверить лог на отсутствие новых ошибок
