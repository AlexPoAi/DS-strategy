Не предлагать платный API (OpenAI, Anthropic Vision) как основной маршрут для задач, которые не требуют работы 24/7.

**Why:** invoice_price_parser.py с Vision API был предложен как production-решение, но упёрся в billing quota. Правильная архитектура: PDF накладные = Mac/вручную/по задаче. VPS 24/7 = только Telegram-бот.

**How to apply:**
- Перед предложением платного API спросить: эта задача нужна 24/7 или разово?
- Разово/по задаче → локальный маршрут (Codex на Mac, ChatGPT через подписку, скрипт без API)
- 24/7/автоматически → тогда обсуждать API, но с явным одобрением пользователя и оценкой стоимости
- VPS 24/7 зарезервирован только для Telegram-бота
