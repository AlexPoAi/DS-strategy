# Жёсткие различения (Hard Distinctions)

> **ЗАПРЕЩЕНО путать эти пары.** Критические онтологические ошибки.

## 1. Система ≠ эпистема

**Тест:** Можно ли указать на границу в пространстве? Есть ли владелец, входы/выходы?

❌ "Система знаний", "система типов", "система мышления"
✅ Экзокортекс (конкретная система), типизация, системное мышление (метод)

## 2. Метод ≠ Инструмент

**Тест:** Можно ли описать метод без названия инструмента?

❌ "Я делаю Toggl"
✅ "Я веду учёт времени в Toggl"

## 3. Рабочий продукт ≠ описание

**Тест:** Можно ли передать это другому человеку без слов?

❌ "У меня есть стратегия" (без документа)
✅ Strategy.md (артефакт)

## 4. Учёт ≠ Планирование

| Учёт | Планирование |
|------|--------------|
| Что было (past) | Что должно быть (future) |
| Дескриптивно | Прескриптивно |

## 5. Роль ≠ Агент ≠ Инструмент

❌ "Стратег — отдельный агент"
✅ Стратег — роль (R1), Claude — holder (исполнитель)

❌ "Бот — агент"
✅ Бот — инструмент (I1), играет роль Проводника (R13)

## 6. Агентская роль ≠ Функциональная роль

| Агентская роль | Функциональная роль |
|----------------|---------------------|
| Стратег, Архитектор | Бариста, Повар |
| Работает с системой | Работает в системе |

## 7. Метод ≠ Работа

**Метод** = описание (U.MethodDescription)
**Работа** = выполнение (U.Work)

❌ "Метод приготовления капучино" = процесс
✅ Метод = спецификация, Работа = конкретное приготовление

## 8. Описание метода ≠ Сценарий

**Описание метода:** Входы → Выходы (внутри BC)
**Сценарий:** Межсистемное взаимодействие (владелец меняется)

## 9. Сервис ≠ Метод

**Сервис** = точка входа (accessSpec)
**Метод** = спецификация работы

## 10. Процесс ≠ Метод

**Процесс (L-PROC):** Житейский термин (полисемичен)
**Метод (U.MethodDescription):** FPF-точный термин

## 11. Репозиторий ≠ Система

**Репозиторий** = хранилище описаний
**Система** = объект с 4D-границами

## 12. Pack ≠ Downstream

**Pack** = source-of-truth (вторые принципы)
**Downstream** = производное (третьи принципы)

Pack меняется → Downstream обновляется

## 13. Bounded Context ≠ Модуль

**BC** = семантические границы (UL, термины)
**Модуль** = технические границы (код, файлы)

## 14. Различение ≠ Определение

**Различение** = пара (A ≠ B)
**Определение** = одиночное (A = X)

## 15. Холон ≠ Компонент

**Холон** = целое И часть (автономия + интеграция)
**Компонент** = только часть
