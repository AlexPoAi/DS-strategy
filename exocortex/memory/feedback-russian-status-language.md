When presenting exocortex status, session-open checks, health summaries, and operator-facing messages, use fully Russian terminology and avoid slipping into English labels like warning, critical, or info.

**Why:** The user wants the system to feel native and immediately readable; English operational terminology breaks comprehension and weakens trust in the interface.

**How to apply:** Use Russian status wording in user-facing outputs and prefer visible colour/state cues such as зелёный / жёлтый / красный with clear symbols so the state is easy to scan both in chat-style interfaces and in terminal outputs.
