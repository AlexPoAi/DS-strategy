When working in `DS-strategy`, prefer short structured cards over long continuous prose.

**Why:** Long text walls are hard for the user to scan during strategy sessions, weekly planning, and work-product coordination. Short cards make the role, task, status, and next step immediately visible.

**How to apply:**
- For strategy sessions, ritual agreement, weekly reviews, weekly planning, backlog triage, notes review, and work-product open/close flows, answer in **short cards** by default.
- Use one card per meaning block instead of one long paragraph.
- Default card fields:
  - `Агент`
  - `Работа`
  - `РП`
  - `Метод`
  - `Оценка`
  - `Статус`
  - `Следующий шаг`
- Free-form prose is still allowed for tiny answers or when the user explicitly asks for plain continuous text.
