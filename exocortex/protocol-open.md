# Протокол Open (WP Gate + Ритуал)

> **Триггер:** Любое задание от пользователя.

---

## WP Gate (блокирующее)

**ОБЯЗАТЕЛЬНО. ДО ЛЮБОГО ДЕЙСТВИЯ.**

1. Прочитать MEMORY.md → секция «РП текущей недели»
2. Совпадает с РП? → Да: п. 3. Нет: СТОП.

**ЗАПРЕЩЕНО до проверки:** читать код, исследовать, планировать.

### Не совпадает — СТОП

1. «Этой задачи нет в плане на неделю.»
2. Вывести таблицу РП из MEMORY.md
3. Спросить: артефакт, формулировка РП, репо, бюджет
4. Записать в три места:
   - `MEMORY.md` → таблица РП
   - `DS-strategy/current/WeekPlan W{N}...` → таблица РП
   - `DS-strategy/inbox/WP-{N}-{slug}.md` → context file (если ≥2h)
5. Только после записи — начать

### Исключения (РП не нужен)

- Задачи ≤15 мин
- Вопросы без изменений файлов
- Экстренные баг-фиксы («срочно»)

**Эскалация:** Если исключение перерастает во второе задание → *«Это РП. Записать?»*

## Синхронизация контекста (блокирующее)

**После WP Gate, перед Ритуалом.**

1. Прочитать `memory/project-brain.md`
2. Прочитать `DS-strategy/current/SESSION-CONTEXT.md`
3. Сравнить с `MEMORY.md` (секция "РП текущей недели")
4. Если расхождения → обновить `MEMORY.md`

**Шаг 1.** Объявить:
