# Правила работы по типам репозиториев

## 3 типа репозиториев

| Тип | Подтип | Что содержит | Source-of-truth | Кто создаёт |
|-----|--------|-------------|-----------------|-------------|
| **Base** | Принципы | ZP, FPF, SPF | Да | Платформа |
| **Base** | Форматы | FMT-* (протоколы структуры) | Да (для формата) | Платформа |
| **Pack** | — | Доменные знания (вторые принципы) | Да | Пользователь |
| **DS** | instrument / governance / surface | Производные от Pack | Нет | Пользователь |

> Base = платформа. Pack = source-of-truth. DS = производное.

---

## Base (платформа)

**Принципы (ZP, FPF, SPF):** Нулевые принципы, фреймворки корректности.
**Форматы (FMT-*):** Протоколы структуры репо.

**Можно:** Уточнять формулировки, обновлять спецификации.
**Нельзя:** Доменное содержание (→ Pack), дидактику (→ DS/surface).

---

## Pack (source-of-truth)

**Содержит:** Формализованные знания домена, методы, work products, failure modes, различения.

**Можно:** Добавлять/изменять знание домена, методы, WP, FM.

**Нельзя:**
- Дидактику (шаги, уроки)
- Путать метод с инструментом
- Хранить планы/сроки/роли (→ Governance)

**Lint:** Проверяй process-lint перед коммитом.

---

## DS (производное)

> DS НЕ source-of-truth. Знание берётся из Pack.

| Подтип | Можно | Нельзя |
|--------|-------|--------|
| `instrument` | Код, тесты, документация | Менять source-of-truth |
| `governance` | Планы, реестры, решения | Менять предметное знание |
| `surface` | Курсы, гайды, маршруты | Создавать знание |

### Правило минимума в DS/instrument

Репо агентов/ботов = код + конфиги + CLAUDE.md (привязка к Pack) + README.

Архитектурные решения → Pack цифровой платформы.

---

## Fallback Chain

DS (3-й) → Pack (2-й) → Base.Принципы (SPF → FPF → ZP)
