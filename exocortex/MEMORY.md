> **БЛОКИРУЮЩЕЕ ПРАВИЛО.** Это правило НЕ МОЖЕТ быть проигнорировано ни при каких обстоятельствах. Нарушение = немедленная остановка работы.

**Алгоритм (выполнять в точном порядке):**

1. **СТОП** — не читать код, не исследовать, не планировать, не запускать команды
2. **Стартовый экран экзокортекса:** показать русский экран состояния среды и статус «мозга экзокортекса»
   - 🟢 зелёный → можно открывать день
   - 🟡 жёлтый → только после явного подтверждения пользователя
   - 🔴 красный → обычное открытие дня ЗАПРЕЩЕНО до устранения проблемы
3. **WP Gate:** Проверить MEMORY.md → секция "РП текущей недели"
4. **Задача есть в плане?**
   - ДА → Ритуал согласования → подтверждение → начать
   - НЕТ → Вывести таблицу РП → спросить параметры → записать в 3 места → ритуал → начать

**Ритуал согласования (6 шагов — ОБЯЗАТЕЛЬНО):**
1. Роль — кто выполняет (агент/пользователь/оба)
2. Работа — что именно делаем
3. РП — номер и название
4. **Агент — какого агента нанять?** → открыть `DS-agent-workspace/agency/REGISTRY.md` → найти профильного → объявить: «Нанимаю [Имя]» или «Агента нет → работаю по умолчанию, фиксирую в GAPS.md»
5. Оценка — бюджет времени
6. Модель — Haiku→Sonnet fallback (Opus запрещён)

**Исключения (РП не нужен):**
- Задачи ≤15 мин
- Вопросы без изменений файлов
- Экстренные баг-фиксы («срочно»)

**Если исключение перерастает во второе задание → "Это РП. Записать?"**

**Примеры НАРУШЕНИЙ (немедленная остановка):**
- ❌ Начал читать файлы до ритуала
- ❌ Начал запускать команды до подтверждения
- ❌ Пропустил проверку WP Gate
- ❌ Не спросил параметры РП для новой задачи
- ❌ Проигнорировал красный статус мозга экзокортекса
- ❌ Начал работу без явного подтверждения пользователя

**Последствия нарушения:**
- Работа останавливается немедленно
- Все незафиксированные изменения теряются
- Требуется перезапуск с соблюдением ритуала
