# WP-65 — Перевод агентного 24/7 в Cloud Scheduler как у Церена

## Цель

Перевести контур 24/7 для инженерного экзокортекса с нестабильного VPS OAuth/runtime на cloud scheduler через GitHub Actions, чтобы система работала при выключенном компьютере.

## Почему это делаем

- На VPS auth-контур Claude нестабилен (обрывы сессий, `loggedIn: false` после OAuth).
- У Церена рабочий паттерн 24/7 реализован через GitHub Actions (cloud scheduler) без зависимости от локальной машины.
- Нужно сохранить только продуктовые сервисы на VPS (bot/rag/warehouse), а агентный мониторинг и ночные проверки вынести в облако.

## Область работ

1. Подтвердить готовность `cloud-scheduler.yml` в `DS-strategy`.
2. Проверить наличие необходимых GitHub Secrets/Variables.
3. Запустить ручной прогон workflow и верифицировать отчёт.
4. Зафиксировать cutover:
   - cloud scheduler = основной 24/7 контур для инженерного мониторинга;
   - VPS exocortex scheduler = отключён;
   - VPS product services = активны.

## Критерии готовности

- [x] `cloud-scheduler.yml` успешно отрабатывает вручную.
- [x] Ночной cron активен.
- [x] Telegram-уведомление от cloud scheduler приходит (если секреты заданы).
- [x] Решение зафиксировано в инженерной хронике.

## Риски

- Отсутствие LLM-задач в cloud scheduler (базовый health/backup контур) — это ожидаемо в данном этапе.
- Возможны ограничения доступа к приватным репозиториям при расширении охвата без PAT.

## Примечание

Этот РП закрывает именно стабильность 24/7 инфраструктуры. Расширение облачного контура на более сложные агентные сценарии будет отдельным РП.
