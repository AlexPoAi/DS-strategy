# WP-95 Phase 1 Scope v2 — стартовый scope нового менеджера после материалов Жанны

## Зачем этот артефакт

После стратегической сессии W18 `WP-95` снова активен. Цель этой карточки — выбрать первый реальный рабочий шаг недели по переходу `Жанна -> новый менеджер`, не раздувая роль до полного операционного менеджера сети.

## Решение на сейчас

Первым шагом недели по `WP-95` считаем не `Phase 2`, а уточнение `Phase 1`.

Рабочая формулировка:

> Найти и запустить локального учётно-складского менеджера, который сначала берёт повторяемый складско-учётный контур, а не всю роль Жанны.

## Что подтверждено материалами Жанны

| Блок | Evidence | Решение |
|---|---|---|
| `Saby / касса` | скринкасты по корректировке составной позиции, добавлению/удалению товара, добавлению составного товара | входит в `Phase 1`, но нужен короткий SOP/checklist |
| Отчётность | скринкасты по отчёту продаж, выгрузке каталога, выручке | входит частично: только регулярные отчёты и prepared uploads, не вся управленческая отчётность |
| Складской учёт | папки `Отчетность` и `Инвентаризация`; `Инвентаризация` пока пустая | входит в `Phase 1`, но запуск без инструкции по инвентаризации рискованный |
| Google Drive | handoff-папка уже создана и наполняется | входит только как intake/операционные документы для `Phase 1`, не как полная перестройка Drive |
| Старый role-contract | `55 000 руб.` за `160-168 ч/мес`, пн-пт 09:00-17:00, отчётность и стратегические сессии | это contract Жанны, не автоматический contract нового Phase 1 менеджера |

## Стартовый scope нового человека

### Входит

1. Остатки по точкам.
2. Инвентаризация.
3. Накладные и приход.
4. Базовая работа с поставщиками по складскому циклу.
5. `Saby` в agreed scope.
6. `Честный знак`.
7. Недостачи и расхождения.
8. Загрузка подготовленных отчётов/файлов в нужные папки Google Drive.
9. Ведение списка открытых хвостов по своему контуру.

### Не входит на старте

1. Полный Telegram-контур сети.
2. Кухня как административный owner.
3. Найм / Avito / первичный HR-фильтр.
4. Полный документооборот компании.
5. Полная перестройка Google Drive.
6. Роль универсального помощника руководителя.
7. Вся текущая роль Жанны по старому contract.

## Compensation conflict

Есть конфликт между двумя слоями:

- старая гипотеза transition: `35 000 / 35 000`;
- новый evidence из role-contract Жанны: `55 000 руб.` за `160-168 ч/мес`.

Вывод:

- `55 000` описывает более широкую роль Жанны, а не узкий `Phase 1`;
- `35 000` может оставаться ориентиром для нового человека только если scope действительно ограничен складско-учётным ядром;
- нельзя обещать кандидату полный операционный scope за `35 000`;
- нельзя снижать роль Жанны до `35 000` без отдельного разговора о том, какие обязанности реально сняты.

## Что нужно добрать у Жанны

Короткий список вопросов:

1. Где инструкция / видео по инвентаризации?
2. Где инструкция по `Честному знаку`?
3. Где правила по недостачам и спорным расхождениям?
4. Где supplier-routine: кому писать, по каким вопросам, какие поставщики критичны?
5. Нужно ли нам делать текстовые SOP из каждого скринкаста?
6. Какая версия role-contract сейчас считается актуальной?
7. Какие задачи Жанна точно хочет снять с себя первыми?

## Что нужно подготовить для кандидата

Перед первым разговором с кандидатом нужен короткий pack:

1. Роль в 5 строк.
2. Список `входит / не входит`.
3. График и формат связи.
4. Оплата как `Phase 1`, без обещания full manager role.
5. Первые 2 недели onboarding.
6. Что считается успешным первым cycle.

## Verdict

`WP-95` на этой неделе должен идти через `Phase 1 Scope v2 + candidate pack`, а не через широкий `Phase 2`.

Первый рабочий результат недели:

- утвердить стартовый scope;
- добрать недостающие инструкции у Жанны;
- обновить текст вакансии / candidate duty map под `Phase 1 Scope v2`;
- только после этого возвращаться к поиску и первому разговору с кандидатом.

## Следующий bounded step

Собрать `candidate pack v2`:

- короткое описание вакансии;
- обязанности;
- не входит;
- вопросы на первый созвон;
- условия оплаты и тестового периода.
