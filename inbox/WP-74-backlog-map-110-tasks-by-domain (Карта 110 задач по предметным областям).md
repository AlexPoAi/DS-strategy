# WP-74: Карта 110 задач по предметным областям

## Почему открываем

Сегодня стало видно, что текущая система перегружена не только количеством задач, но и отсутствием единой доменной карты.

Сейчас задачи живут сразу в нескольких слоях:
- extractor / notes;
- `INBOX-TASKS.md`;
- weekly carry-over;
- отдельные `WP`.

Пока эти слои не сведены в одну карту, невозможно стабильно работать по правилу:
`один домен = один главный активный РП`.

## Цель

Собрать полный реестр задач и заметок, которые реально участвуют в стратегическом управлении, и присвоить каждой записи предметную область.

## Что должно получиться

Единый стратегический реестр, где у каждой записи есть:
- `ID`
- `Источник`
- `Тип`
- `Предметная область`
- `Поддомен`
- `Статус`
- `Приоритет`
- `Есть ли уже WP`
- `Следующий шаг`

## Какие слои нужно свести

1. Extractor / notes layer
2. `INBOX-TASKS.md`
3. Активные `WP-*`
4. Weekly carry-over

## Предварительные домены

- `VK Coffee / Тургенева`
- `VK Coffee / Луговая`
- `VK Coffee / Самокиша`
- `VK Coffee / Кухня`
- `VK Coffee / Маркетинг и рост`
- `VK Coffee / Сервис и управление`
- `Парк`
- `Склад`
- `Инженерный контур`
- `Экзокортекс / агенты / runtime`
- `Рефакторинг репозиториев и knowledge architecture`
- `Личное / стратегия`

## Acceptance

- Есть полный список backlog-единиц для стратегического слоя.
- Каждая единица отнесена к домену.
- Для каждого домена виден один главный активный РП и очередь остальных задач.
- На основе этой карты можно управлять неделей как портфелем доменов, а не хаотичным списком.

2. Собрать первый truthful реестр.
3. Сделать доменную классификацию.
4. На основе этого пересобрать weekly portfolio rule.
