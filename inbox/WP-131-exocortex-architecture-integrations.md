# WP-131: Экзокортекс — архитектура и интеграции

> **Задача:** Сделать архитектуру видимой и простой. Расширить Telegram-бот. Найти и запустить новые автоматизации.

## Контекст

Экзокортекс работает как чёрный ящик:
1. Нет архитектурной карты — кажется мусором, неясно что живое vs архив
2. Telegram-бот неполный — только логирование и отчёты, нет RAG, поиска, оркестратора
3. Не исследованы возможности автоматизации бизнес-процессов (Codex/Claude)
4. Google Drive не переработан — может быть инструкции для автоматизации

## План

### Блок 1: Карта архитектуры (1-1.5h)

**Результат:** Диаграмма или инфографика, которая показывает: где какие агенты, как связаны, какие данные куда текут.

- [ ] Перечислить ВСЕ компоненты: agents (стратег, экстрактор, финансист, кладовщик), системы (Telegram, Obsidian, Google Drive, Pack), хранилища (MEMORY, WP-REGISTRY, captures, коллекции)
- [ ] Определить связи: кто пишет в кого, какой сигнал, тип интеграции (sync, event, cron, manual)
- [ ] Нарисовать диаграмму (Miro, draw.io или текстовая ASCII в Markdown)
- [ ] Добавить в DS-strategy/docs/ как `ARCHITECTURE.md`

**Критерий готовности:** По диаграмме очевидно: что живо, что архив, какие данные как текут.

### Блок 2: Audit legacy (0.5-1h)

**Результат:** Реестр старых маршрутов и неиспользуемых папок. Рекомендация: что удалить, что оставить.

- [ ] Провести grep по коду: какие переменные, пути, функции не используются в текущем дне (git log за 1 месяц)
- [ ] В Google Drive: какие папки не обновлялись >месяца
- [ ] В Obsidian: какие заметки не изменялись >месяца (или заархивированы)
- [ ] Составить список с рекомендациями: перечитать/оставить в архиве/удалить

**Критерий готовности:** WP-131-legacy-audit.md содержит рекомендации по cleanup.

### Блок 3: Telegram-бот (1.5-2h)

**Результат:** Telegram-бот умеет: RAG-поиск по базе знаний, поиск по документам, читать/писать чаты, публиковать посты, управлять агентами.

#### Подблок 3.1: RAG-поиск (0.5h)

- [ ] Установить инструмент: LangChain + embeddings (OpenAI / local)
- [ ] Загрузить в векторную БД: Pack'и, WP-REGISTRY, MEMORY, captures
- [ ] Добавить endpoint в Telegram-бот: `/search <query>` → returns top 3 matches

#### Подблок 3.2: Чтение/запись чатов (0.5h)

- [ ] Telegram Bot API: получить доступ к истории чатов (если приватный чат с ботом)
- [ ] Добавить команды: `/read-chat <chat_id>` (выгрузить последние N сообщений)
- [ ] Добавить команды: `/post <message>` (отправить в выбранный чат)

#### Подблок 3.3: Оркестратор агентов (0.5h)

- [ ] Создать маршрут: Telegram-команда → определить целевой агент → запустить
- [ ] Примеры команд:
  - `/run warehouse` → запустить warehouse-demand-analyst
  - `/run finance` → запустить vk-finance-analyst
  - `/audit [pack-name]` → запустить extractor на аудит Pack'а
- [ ] Результат отправляется в Telegram

**Критерий готовности:** Пользователь может `/search фраза` в Telegram и получить результаты; может запустить агента через Telegram.

### Блок 4: Автоматизации кофейни (1-1.5h)

**Результат:** Выявлено ≥3 процессов, которые можно автоматизировать через Codex/Claude.

- [ ] Провести интервью: какие ручные процессы сейчас (готовка, закупки, отчёты, расписание)?
- [ ] Для каждого: описать IPO (input, process, output)
- [ ] Для каждого: предложить автоматизацию (ручной сценарий → код → интеграция)
- [ ] Выбрать топ-3, создать простые скрипты/функции
- [ ] Примеры:
  - Генерация ежедневного меню по предыдущим выборам
  - Закупочная ведомость по остаткам (обратная связь от кладовщика)
  - Отправка напоминаний баристам по расписанию
  - Weekly финансовый дайджест вместо ручной сборки

**Критерий готовности:** В Codex есть 3+ скрипта, которые автоматизируют кофейню. Хотя бы один работает в production.

### Блок 5: Google Drive (0.5-1h)

**Результат:** Переговор с Drive, выявлены источники для автоматизации.

- [ ] Для каждого файла: определить кто читает, как используется, есть ли потенциал для автоматизации
- [ ] Создать реестр: [Документ | Текущее использование | Потенциал автоматизации]

**Критерий готовности:** WP-131-drive-audit.md содержит реестр и рекомендации.
