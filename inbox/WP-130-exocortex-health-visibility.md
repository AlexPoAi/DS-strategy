# WP-130: Экзокортекс — видимость и здоровье

> **Задача:** Сделать видимыми статус и проблемы работающих агентов. Восстановить Obsidian-интеграцию. Организовать автоматическую архивацию документов.

## Контекст

Экзокортекс накопил проблемы по видимости:
1. Нет health-отчёта в Telegram — неизвестно когда агент упал, когда был последний успех
2. Obsidian-интеграция непрозрачна — агент читает заметки? Конвертирует в captures?
3. Документы кладовщика и финансиста остаются в «Новые документы» (не переходят в «Обработано»)
4. Реальная готовность агентов неизвестна (финансист и кладовщик в каком статусе?)

## План

### Блок 1: Health-отчёт в Telegram (1-1.5h)

**Результат:** Каждый час/6 часов в Telegram идёт отчёт об агентах: живы ли, какой статус последнего run, есть ли ошибки.

- [ ] Определить метрики health (uptime, last_success_time, error_count, last_error)
- [ ] Создать/обновить скрипт health_check.py в DS-agent-workspace
- [ ] Интегрировать с Telegram-ботом (new endpoint /health или daily report)
- [ ] Настроить scheduler (cron или launchd)
- [ ] Smoke-test: запустить вручную, проверить сообщение в Telegram

**Критерий готовности:** Telegram получает отчёт, в нём видны ошибки если есть.

### Блок 2: Obsidian + архивация (1-1.5h)

#### Подблок 2.1: Obsidian-интеграция (0.5h)

- [ ] Прочитать текущее состояние: какой скрипт читает Obsidian? (поиск в DS-agent-workspace)
- [ ] Проверить path: Obsidian → captures.md (синхронизация работает?)
- [ ] Если не работает — востановить path или создать новый
- [ ] Если есть двусторонняя интеграция, проверить что обе стороны актуальны

**Критерий готовности:** Новая заметка в Obsidian → Через X минут появляется в DS-strategy/inbox/captures.md

#### Подблок 2.2: Архивация документов Drive (1h)

- [ ] Найти скрипт warehouse-biweekly-processor.sh или аналог (обрабатывает Новые документы)
- [ ] Добавить этап архивации: после обработки в Pack → переместить файл из «Новые документы» → «Обработано»
- [ ] Проверить folder IDs (возможны сбои с hard-coded путями)
- [ ] Настроить автоматизацию: по расписанию или по триггеру обработки

**Критерий готовности:** После того как Document был обработан агентом → автоматически переместился в Drive в «Обработано»

### Блок 3: Готовность агентов (0.5h)

- [ ] Прочитать WORKPLAN финансиста и кладовщика
- [ ] Определить текущий статус каждого (MVP / beta / production)
- [ ] Документировать: какой функционал work, какой missing, какие blocker'ы
- [ ] Записать в VK-offee/PACK-warehouse/01-pack-manifest.md § Агенты (или новый файл WH.READINESS.md)

**Критерий готовности:** В Pack явно задокументирована готовность обоих агентов.
