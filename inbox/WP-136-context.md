# WP-136 context

## Церен / upstream

Upstream Церена допускает launchd intake-потоки и протокольный close только через согласованный ACTIVE-WP и scope.

## Наш идеал

Ежедневный intake исчезающих заметок работает автоматически, с догоном при старте сессии, и граф Obsidian остаётся связным.

## Наш факт

Добавлен новый intake-агент и скрипт, выполнен ручной прогон, добавлены базовые связи по Парку.

## Разрыв / Gap

До фикса не было отдельного intake-контура Obsidian и не было стандарта граф-связей для ключевых заметок.

## Решение по текущему фиксу

Минимальный bounded фикс: отдельный `obsidian-fleeting-intake.sh` + launchd-agent + scoped обновление ключевых заметок.
