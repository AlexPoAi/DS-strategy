# WP-98 — Нормализация SKU склада и доведение decision board

## Зачем

После manager-ready рефактора складского отчёта остались два живых дефекта:
- в отчёте встречается битый SKU вида `???? ... 10 шт`
- `WH.SESSION.001` нужно удерживать как decision board для руководителя, а не давать скатываться обратно в документный шум

Дополнительно уточнён доменный контекст:
- дрипы `Субмарина` должны отличаться от `Тэйсти Кофе`
- десерты и кухня не идут как direct reorder владельцу; только аналитика цен и продаж

## Целевой результат

1. Убрать битые SKU из manager-facing слоя
2. Развести supplier routing для `Тэйсти Кофе` и `Субмарина`
3. Закрепить `WH.SESSION.001` как decision board
4. Зафиксировать следующий отдельный слой: OCR-агент по PDF-накладным (`WP-97`)

## Truthful статус

Открыт как отдельный bounded slice после прямого замечания владельца по качеству warehouse-отчёта.

## Что сделано

1. Удержан manager-facing слой без битых `????` SKU и мусорной нормализации.
2. `WH.SESSION.001` переведён из списка позиций в action-board по поставщику:
   - одно действие;
   - поставщик;
   - канал;
   - дедлайн;
   - позиции внутри заказа.
3. `WH.REPORT.002` получил новый `Executive Summary`:
   - число критичных SKU;
   - число поставщиков в цикле;
   - главный риск периода;
   - честная пометка про отсутствие `ABC`.
4. Telegram digest и локальный manager-report пересобраны на живых данных.
5. Живой manual-run выполнен, Telegram-отчёт отправлен.

## Truthful результат

`WP-98` как bounded slice завершён.

Что закрыто:
- decision board больше не скатывается в файловый шум;
- supplier-based action layer для срочного заказа materialized;
- верхний summary стал manager-readable.

Что сознательно НЕ закрыто этим РП:
- полноценный OCR по PDF-накладным (`WP-97`);
- достройка `ABC` как входа в текущий цикл;
- полное разведение `Тэйсти Кофе` / `Субмарина` по live invoice evidence для всех drip SKU.

## Ритуальная пометка

В первой попытке был ошибочно поднят временный sidecar-аудитор без согласования.
Это не считается правильным шаблоном исполнения.

Правильный контур для этого РП:
- основной агент: `Кладовщик VK Coffee`
- инженер: только поддержка skill/pipeline/runtime
