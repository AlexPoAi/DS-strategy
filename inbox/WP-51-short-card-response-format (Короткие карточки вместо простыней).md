# WP-51: Короткие карточки вместо простыней

## Контекст

Пользователь явно зафиксировал устойчивое предпочтение по формату взаимодействия:

- ответы должны быть короткими;
- ответы должны быть в формате чёткой карточки;
- длинные простыни текста мешают работе;
- развёрнутый режим нужен только по отдельному запросу.

Это не локальная просьба на одну сессию, а постоянное правило взаимодействия.

## Новое базовое правило

По умолчанию все ответы давать в коротком карточном формате.

Базовый шаблон:

1. `Что это`
2. `Что это значит`
3. `Следующий шаг`

Допускается сокращённый формат:

- `Вывод`
- `Действие`

## Ограничения

- Не разворачивать длинные объяснения без прямого запроса пользователя.
- Не превращать простой вывод в эссе.
- Не писать “простыни” там, где можно дать 2-4 ясных пункта.

## Где зафиксировано

- `current/USER-PREFERENCES.md`
- `current/SESSION-CONTEXT.md`

## Критерий выполнения

- В новых ответах по умолчанию используется короткая карточная форма.
- Длинный режим включается только по явному запросу пользователя.
