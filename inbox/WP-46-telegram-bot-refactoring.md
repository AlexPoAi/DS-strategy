# WP-46: Рефакторинг Telegram-бота VK-offee — сценарии, меню, архитектура

## Цель

Провести полный рефакторинг Telegram-бота VK-offee:
1. Спроектировать сценарии использования для сотрудников кофейни
2. Разработать меню бота (кнопки, команды, навигация)
3. Обновить архитектуру для расширения функционала
4. План реализации с приоритетами

## Контекст

**Текущее состояние:**
- Бот работает на VPS `72.56.4.61` (Timeweb Cloud)
- RAG API с ChromaDB: 436 документов (7 Pack + 122 Saby-карточки)
- Keyword-routing для накладных Saby работает ✅
- Базовый функционал: `/start` + текстовый поиск
- Интеграция: Claude API через прокси + локальные embeddings

**Проблемы:**
- Нет структурированного меню — только текстовый ввод
- Нет сценариев под роли (бариста, повар, менеджер, шеф)
- Нет быстрых действий (кнопки, inline-клавиатура)
- Поиск работает, но UX не оптимизирован для мобильного

**Возможности:**
- База знаний VK-offee: 7 PACK (bar, kitchen, service, management, hr, park, saby-reports)
- RAG находит накладные, рецепты, процессы, документы
- Можно добавить персонализацию по ролям
- Можно интегрировать с внешними системами (Saby API, Google Drive)

## Часть 2: Сценарии использования

### Сценарий 1: Бариста — Рецепт напитка

**Контекст:** Бариста забыл пропорции капучино 0.4л
**Действие:**
1. Открывает бота → кнопка "☕ Рецепты"
2. Выбирает "Капучино" из списка популярных
3. Видит карточку: эспрессо 60мл, молоко 340мл, температура 65°C

**Альтернатива (текстовый поиск):**
- Пишет "капучино 0.4" → бот находит рецепт

**Ценность:** Быстрый доступ к рецептам без поиска в папках

---

### Сценарий 2: Повар — Срок годности заготовки

**Контекст:** Повар нашёл контейнер с заготовкой, нужно проверить срок
**Действие:**
1. Открывает бота → кнопка "🍳 Кухня"
2. Выбирает "Сроки годности"
3. Вводит название заготовки или сканирует QR-код
4. Видит: дата изготовления, срок хранения, статус (годен/просрочен)

**Ценность:** Предотвращение использования просроченных заготовок

---

### Сценарий 3: Менеджер — Накладная от поставщика

**Контекст:** Менеджер принимает поставку, нужно проверить накладную
**Действие:**
1. Открывает бота → кнопка "📦 Накл��дные"
2. Выбирает поставщика из списка (Хорика, МФуд, Дмитрий)
3. Видит последние 5 накладных с дата��и и суммами
4. Открывает нужную → детали товаров, цены, изменения

**Альтернатива:**
- Пишет "накладная Хорика" → бот находит все накладные

**Ценность:** Быстрая проверка цен и товаров при приёмке

---

### Сценарий 4: Официант — Аллергены в блюде

**Контекст:** Гость спрашивает про аллергены в боуле
**Действие:**
1. Открывает бота → кнопка "🍽️ Меню"
2. Выбирает "Боул с курицей"
3. Видит состав + маркировку аллергенов (глютен, молоко, орехи)

**Ценность:** Быстрый ответ гостю, безопасность

---

### Сценарий 5: Шеф — Аналитика продаж

**Контекст:** Шеф хочет посмотреть топ-5 продаж за неделю
**Действие:**
1. Открывает бота → кнопка "📊 Аналитика"
2. Выбирает период (неделя/месяц)
3. Видит топ-5 позиций с выручкой и количеством

**Ценность:** Быстрый доступ к данным без входа в Saby

## Часть 3: Архитектура меню

### Главное меню (ReplyKeyboardMarkup)

```
┌─────────────────────────────────┐
│  ☕ Рецепты    │  🍳 Кухня       │
├─────────────────────────────────┤
│  📦 Накладные  │  🍽️ Меню        │
├─────────────────────────────────┤
│  📊 Аналитика  │  ❓ Поиск       │
└─────────────────────────────────┘
```

**Персонализация по ролям:**
- Бариста видит: Рецепты, Меню, Поиск
- Повар видит: Кухня, Меню, Поиск
- Менеджер видит: Накладные, Аналитика, Поиск
- Шеф видит: всё

### Подменю "☕ Рецепты" (InlineKeyboardMarkup)

```
Популярные напитки:
[Капучино] [Латте] [Американо]
[Флэт Уайт] [Раф] [Матча]

[🔍 Поиск по названию]
[⬅️ Назад в главное меню]
```

### Подменю "📦 Накладные"

```
Поставщики:
[Хорика] [МФуд] [Дмитрий]
[Смакотерия] [Умами]

[📅 По дате] [🔍 Поиск]
[⬅️ Назад]
```

## Часть 4: Техническая архитектура

### Модульная структура

```
telegram-bot/
├── bot.py              # Главный файл, роутинг
├── handlers/           # Обработчики команд
│   ├── recipes.py      # ☕ Рецепты
│   ├── kitchen.py      # 🍳 Кухня
│   ├── invoices.py     # 📦 Накладные
│   ├── menu.py         # 🍽️ Меню
│   ├── analytics.py    # 📊 Аналитика
│   └── search.py       # ❓ Поиск (RAG)
├── keyboards/          # Клавиатуры
│   ├── main_menu.py
│   └── inline_menus.py
├── services/           # Бизнес-логика
│   ├── rag_client.py   # RAG API клиент
│   └── user_roles.py   # Управление ролями
└── config.py           # Конфигурация
```

### Улучшения RAG

**1. Контекстный роутинг:**
- Бариста → приоритет PACK-bar
- Повар → приоритет PACK-kitchen
- Менеджер → приоритет saby-reports + PACK-management

**2. Кэш частых запросов:**
- Топ-10 рецептов → кэш на 1 час
- Последние накладные → кэш на 30 мин

**3. Быстрые ответы (без RAG):**
- "Цена капучино" → прямой lookup в прайс-листе
- "Телефон поставщика" → прямой lookup в справочнике
