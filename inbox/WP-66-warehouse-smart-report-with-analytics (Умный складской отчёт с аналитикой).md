# WP-66: Умный складской отчёт с аналитикой

## Проблемы текущего отчёта

1. **Приходит каждые 6-12ч даже без новых данных** — нет фильтра "только если есть новые карточки"
2. **Ссылки в Telegram не кликабельны** — копируется текст вместо перехода на GitHub
3. **Отчёт пустой по смыслу** — просто список имён карточек, нет аналитики

## Задача

### Часть 1 — Починить частоту и ссылки (быстро)
- Отправлять отчёт ТОЛЬКО если с последнего отчёта появились новые карточки
- Ссылки на карточки → кликабельные GitHub URL вида:
  `https://github.com/alexpoaiagent-sudo/VK-offee/blob/main/PACK-warehouse/02-domain-entities/report-cards/WH.CARD.*.md`

### Часть 2 — Умный аналитический отчёт (основное)
После синка папки Жанны агент должен:

1. **Прочитать остатки** — открыть карточки остатков, извлечь позиции и количества
2. **Прочитать ABC-анализ** — открыть карточки ABC, извлечь категории A/B/C по позициям
3. **Сделать аналитику:**
   - Позиции категории A с низким остатком → срочно заказать
   - Позиции категории C с высоким остатком → не заказывать / сократить
   - Позиции без остатков → критичный алерт
4. **Прислать в Telegram один связный отчёт:**
   ```
   📊 Складской анализ [дата]
   
   🔴 Срочно заказать (A + мало):
   - Позиция X: осталось 2 кг
   
   🟡 Обратить внимание (B + мало):
   - ...
   
   🟢 В порядке:
   - ...
   
   ⚠️ Избыток (C + много):
   - Позиция Y: 50 кг, продаётся редко
   ```

## Acceptance
- [ ] Отчёт приходит только при новых данных
- [ ] Ссылки кликабельны и ведут на GitHub
- [ ] Отчёт содержит реальные позиции с количествами
- [ ] ABC-анализ используется для приоритизации
- [ ] Нет дублей при повторном запуске
