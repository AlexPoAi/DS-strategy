# WP-59: Telegram-бот — роль управляющего и быстрый доступ к дневному/недельному отчёту

## Цель

Добавить в Telegram-бота VK-offee отдельный управленческий сценарий:

- выбор роли `Управляющий`;
- вход в раздел `Отчёты`;
- две быстрые кнопки:
  - `Дневной отчёт`
  - `Недельный отчёт`

## Контекст

Пользователь получил в Telegram сильный недельный отчёт со ссылкой на GitHub и хочет, чтобы такой доступ был не только из уведомления, но и прямо из Telegram-бота.

Дополнительно появилась правильная архитектурная идея:

- не выносить отчёты в общий корень меню для всех;
- а встроить их в роль `Управляющий кофейней`.

Это лучше соответствует реальным сценариям:

- бариста/повару не нужны инженерные и стратегические отчёты экзокортекса;
- менеджеру/управляющему нужен быстрый доступ к operational summary.

## Что уже найдено

- weekly report link уже существует в `DS-strategy/current/WeekPlan W15 2026-04-06.md`
- дневные отчёты уже материализуются в scheduler reports
- Telegram-бот живёт в:
  - `VK-offee/telegram-bot/bot.py`
- контекст бота уже описан в:
  - `DS-strategy/inbox/WP-46-telegram-bot-refactoring.md`
  - `DS-strategy/inbox/WP-48-telegram-bot-testing.md`

## Предлагаемая UX-схема

### Вариант 1 — предпочтительный

`Роль` → `Управляющий` → `Отчёты` → `Дневной отчёт / Недельный отчёт`

Плюсы:

- логически чисто;
- не засоряет главное меню;
- масштабируется под дополнительные менеджерские функции.

### Вариант 2 — быстрый shortcut

Сразу отдельная кнопка `📊 Отчёты` в корневом меню.

Минусы:

- смешивает operational role UX с общим поисковым меню;
- хуже для дальнейшей ролевой модели.

## Предварительный verdict

Правильный путь:

- делать через роль `Управляющий`;
- отчёты показывать как submenu;
- для MVP достаточно давать ссылки:
  - на последний дневной отчёт;
  - на последний недельный отчёт.

## Что сделать

1. Найти canonical source для:
   - последнего дневного отчёта;
   - последнего недельного отчёта.
2. Добавить в бота role-aware меню для `Управляющего`.
3. Добавить кнопки:
   - `📅 Дневной отчёт`
   - `🗓️ Недельный отчёт`
4. Проверить, как бот отдаёт ссылки:
   - GitHub link;
   - или короткий summary + ссылка.
5. Протестировать UX на телефоне.
