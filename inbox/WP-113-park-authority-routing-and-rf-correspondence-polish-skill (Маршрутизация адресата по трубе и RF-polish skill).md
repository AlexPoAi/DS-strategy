# WP-113 — Маршрутизация адресата по трубе и RF-polish skill

## Зачем открыт

Нужно не просто дописать письмо в администрацию, а сначала:

1. truthfully решить, правильный ли адресат выбран для следующего запроса по трубе;
2. усилить Park-agent skill так, чтобы он доводил официальные документы до
   реально `print-ready / ready-to-send` состояния по нормам деловой переписки РФ.

## Граница slice

Внутри этого WP:

- оценка адресата для второго официального запроса по трубе;
- формализация правила, что районный уровень запрашивается не про
  `кадастровый учёт трубы`, а про:
  - балансодержателя;
  - правовой статус сети;
  - компетентный орган;
  - переадресацию по подведомственности;
- создание отдельного support-skill для Park-agent:
  - шапка справа;
  - отсутствие пустых реквизитов;
  - clean-formatter под официальный документ РФ;
  - приёмка документа до `под ноль`.

Вне этого WP:

- не отправляем новое письмо;
- не переводим письмо в `print-ready`;
- не даём юридическое заключение по существу.

## Routing verdict

На текущем factual контуре:

- первый адресат уже выбран правильно: `ГУП РК "Вода Крыма"`;
- второй адресат остаётся разумным: `Администрация Бахчисарайского района`
  на районном уровне, а не сельсовет;
- но предмет второго письма должен быть сужен.

Правильный предмет обращения в район:

1. Есть ли сведения о балансодержателе / собственнике сети.
2. Муниципальная ли это сеть или есть признаки бесхозяйной инженерной сети.
3. Кто является компетентным органом по:
   - обследованию;
   - выносу / переносу;
   - сохранению сети;
   - согласованию действий в отношении неё.
4. Если вопрос не их — переадресовать по подведомственности или письменно
   указать точного адресата.

Неправильная формулировка для этого письма:

- запрос про “кадастровый учёт трубы” как основной предмет.

Это может быть отдельным последующим треком через `Росреестр/ЕГРН`, если
обнаружится, что труба оформлена как самостоятельный объект.

## Acceptance

Slice считается успешным, если:

1. адресат и предмет письма в район различены truthfully;
2. у Park-agent появляется отдельный RF-polish skill;
3. skill зафиксирован и в агентстве, и в Pack-agent-skills.

## Truthful status

После этого WP следующим отдельным slice должен идти уже не анализ, а
доведение письма в администрацию до `print-ready`.
