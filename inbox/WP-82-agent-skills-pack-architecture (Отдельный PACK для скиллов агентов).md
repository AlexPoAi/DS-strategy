# Контекст

Скиллы агентов сейчас распределены фрагментарно. Нужен отдельный доменный `PACK`, чтобы:
- централизованно хранить скиллы;
- версионировать и масштабировать агентную экспертизу;
- быстро подключать скиллы новым агентам без дублирования.

# Цель

Спроектировать и запустить отдельный `PACK` под скиллы агентов как source-of-truth для skill-контрактов, шаблонов и runbook-ов.

# Scope

1. Спроектировать структуру `PACK-agent-skills` (контракты, сущности, методы, рабочие продукты).
2. Описать формат skill-карточки (входы, выходы, шаги, критерии качества, ограничения).
3. Перенести первые 3-5 ключевых скиллов в новый PACK (включая складского кладовщика).
4. Связать PACK со слоем агентства (`DS-agent-workspace/agency/agents/*`).
5. Зафиксировать процесс обновления и версионирования скиллов.

# Acceptance

1. В репозитории есть новый `PACK` со структурой и manifest.
2. Есть единый шаблон skill-карточки.
3. Минимум 3 скилла перенесены и работают по контракту.
4. Каждый агент в карточке ссылается на skill из PACK, а не на разрозненные инструкции.
