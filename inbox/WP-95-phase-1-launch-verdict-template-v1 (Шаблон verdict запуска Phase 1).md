## Назначение

Этот шаблон нужен, чтобы verdict по запуску `Phase 1` оформлялся одинаково и не сводился к впечатлению “вроде пошло” или “что-то не нравится”.

Использовать:

## Правило verdict

### `success`

Ставим только если одновременно выполнены условия:

1. bounded scope понятен;
2. первый cycle прошёл без критического развала;
3. по ядру нет критичных провалов;
4. Жанна перестала быть скрытым owner;
5. можно удерживать контур без немедленного отката.

### `partial`

Ставим, если:

- есть рабочая база, но ownership ещё нестабилен;
- часть блоков держится, часть требует повторного handoff;
- запуск можно продолжать только в bounded corrective режиме.

### `fail`

Ставим, если:

- роль осталась размытой;
- кандидат не удержал ядро;
- handoff оказался только “на словах”;
- без возврата к Жанне контур сразу разваливается.

## Решение после verdict

### Если `success`

1. Подтверждаем, что `Phase 1` передана.
2. Не расширяем роль мгновенно.
3. Отдельно открываем slice на обсуждение `Phase 2`.

### Если `partial`

1. Не идём в `Phase 2`.
2. Назначаем повторный bounded handoff.
3. Фиксируем, какой именно блок требует починки.

### Если `fail`

1. Останавливаем расширение.
2. Не называем запуск успешным.
3. Решаем отдельно:
   - повторить старт;
   - заменить кандидата;
   - пересобрать модель handoff.
