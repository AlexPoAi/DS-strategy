# Контекст

В `VK-offee` накоплен незакрытый слой задач (менеджмент, маркетинг, аналитика, операционные стандарты). Универсального агента недостаточно: нужен набор предметных агентов с явными доменными границами и skill-протоколами.

# Цель

Сформировать и внедрить агентную линейку VK-coffee, которая стабильно производит полезные артефакты по ключевым доменам.

# Целевая линейка агентов

1. Bar Ops Agent
2. Kitchen Ops Agent
3. Service Ops Agent
4. Marketing & Growth Agent
5. Finance & Inventory Agent

# Scope

1. Контракт каждого агента: входы, выходы, границы, SLA артефактов.
2. Skill-пакеты под домен и структуру `VK-offee`.
3. Шаблоны промптов/чеклистов по каждому домену.
4. Пилотный прогон на живых задачах.
5. Интеграция с Codex Pro image-workflow (визуалы в карточках/доках).

# Acceptance

1. У каждого агента есть собственная карточка/контракт и набор шагов выполнения.
2. Минимум 3 доменных агента прошли live-задачи с полезным артефактом.
3. По маркетингу и менеджменту VK-coffee сформирован actionable план.
4. Все артефакты привязаны к repo-first маршруту и видны новому агенту из контекста.
