# WP-46: Превратить файлы Extractor в рабочие продукты

## Проблема

Extractor создаёт файлы в Pack-репозиториях (VK-offee, PACK-bar, PACK-kitchen и т.д.), но **не создаёт задачи** на их доработку.

Результат: файлы лежат в Pack, но мы не знаем о них → они не попадают в работу.

## Задача

1. **Найти все файлы**, созданные Extractor за последнее время
2. **Проверить каждый файл** — что в нём, насколько он готов
3. **Создать РП для каждого файла** в `DS-strategy/inbox/`
4. **Распределить РП** по приоритетам

## Где искать файлы Extractor

Extractor создаёт файлы в:
- `~/Github/VK-offee/` (все подпапки)
- `~/Github/PACK-bar/`
- `~/Github/PACK-kitchen/`
- `~/Github/PACK-warehouse/`
- `~/Github/PACK-park-development/`

**Как найти:** Файлы с датой создания за последние 2 недели.

## Что делать с каждым файлом

1. Прочитать файл
2. Оценить готовность (draft / partial / ready)
3. Если draft/partial → создать РП на доработку
4. Если ready → оставить как есть

## Формат РП

Для каждого файла создать:

```
WP-XXX: Доработать [название файла]

**Файл:** [путь к файлу]
**Статус:** draft/partial
**Что нужно:**
- [ ] Дополнить контент
- [ ] Проверить структуру
- [ ] Добавить примеры
- [ ] Финализировать

**Бюджет:** 30 мин - 1h
```

## Следующий шаг

После выполнения этого РП → **автоматизировать процесс**:
- Extractor создаёт файл в Pack
- Extractor создаёт РП в inbox
- РП попадает в работу автоматически

## Критерии готовности

- [ ] Найдены все файлы Extractor за последние 2 недели
- [ ] Для каждого draft/partial файла создан РП
- [ ] РП добавлены в `DS-strategy/inbox/`
- [ ] РП распределены по приоритетам
