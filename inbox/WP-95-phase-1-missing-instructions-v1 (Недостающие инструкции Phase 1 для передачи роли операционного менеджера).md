# WP-95 Phase 1 Missing Instructions v1

## Контекст

На `2026-04-27` Жанна уже предоставила:

- Miro-карту должности операционного менеджера;
- PDF/CSV экспорт карты;
- Drive-папку `Операционный менеджер`;
- скринкасты по Saby / кассе;
- скринкасты по отчетности;
- старый документ по роли;
- пустую пока папку `Складской учет / Инвентаризация`.

Рабочий вывод: роль начала проявляться, но пока в формате карты и видео, а не в формате готовой системы для передачи новому человеку.

## Управленческое решение

Мы не заменяем Жанну как человека.

Мы выделяем из текущей работы по роли операционного менеджера повторяемые процессы, которые можно передать новому человеку через:

- карту обязанностей;
- инструкции;
- чеклисты;
- критерии готовности;
- понятные результаты работы.

## Граница Phase 1

В Phase 1 включаем только:

- складской учет;
- накладные;
- инвентаризацию;
- Честный знак;
- остатки;
- отчетность;
- базовую работу с Saby только в рамках этих процессов.

В Phase 1 не включаем:

- Авито и найм;
- обновление всех регламентов;
- широкий Telegram-контур;
- любые поручения директора / шеф-бариста;
- роль “человек на всё”.

## Недостающие инструкции

| Блок | Что нужно получить |
|---|---|
| Инвентаризация | как проводится, какие файлы используются, куда заносится результат, кто проверяет, что делать с расхождениями |
| Накладные | приход, списание, перемещение, электронные накладные, типовые ошибки |
| Честный знак | что именно выводится, как часто, где проверять результат |
| Остатки по складу | как выгружать зерно, бункер, чай, шоколад, куда складывать файл |
| Отчетность | какие файлы обязательны, откуда выгружать, куда загружать, кто смотрит |

## Запрос Жанне

> Жанна, посмотрели карту должности и материалы в папке. Спасибо, уже видна структура работы операционного менеджера: склад, накладные, инвентаризация, Честный знак, отчетность, Saby/касса, Авито и процессы.
>
> Сейчас для передачи части работы новому человеку нам нужно не расширять карту, а дописать инструкции по самым регулярным процессам Phase 1.
>
> В первую очередь нужны:
>
> 1. Инвентаризация: как проводится, какие файлы используются, куда заносится результат, кто проверяет, что делать с расхождениями.
> 2. Накладные: приход, списание, перемещение, электронные накладные, типовые ошибки.
> 3. Честный знак: что именно выводится, как часто, где проверять результат.
> 4. Остатки по складу: как выгружать зерно, бункер, чай, шоколад, куда складывать файл.
> 5. Отчетность: какие файлы обязательны, откуда выгружать, куда загружать, кто смотрит.
>
> По каждому блоку достаточно коротко: что делается, как часто, откуда берем данные, какой результат должен получиться, куда кладем, кто проверяет, какие ошибки бывают.
