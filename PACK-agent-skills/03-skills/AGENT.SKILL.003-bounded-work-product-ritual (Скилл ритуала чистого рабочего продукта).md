# AGENT.SKILL.003 — Скилл ритуала чистого рабочего продукта

## Назначение

Повторяемый контракт работы в живом репозитории через bounded work products,
чистые границы slice и отдельные truthful commits.

## Последовательность

1. Открыть новый bounded `WP`.
2. Явно назначить primary/support агента.
3. Проверить текущее состояние дерева и выделить свой write-set.
4. Реализовывать только в пределах объявленного slice.
5. Закрыть `WP` truthfully:
   - что materialized;
   - что не сделано;
   - какой следующий bounded шаг.
6. Сделать отдельный `commit/push` только по своему scope.

## Жёсткие запреты

- Нельзя начинать правки без открытого РП.
- Нельзя расширять scope по ходу работы без нового согласования.
- Нельзя смешивать чужие хвосты со своим commit.
- Нельзя закрывать `WP` как done, если slice не доведён truthfully.
- Нельзя коммитить runtime-артефакты только потому, что они изменились рядом.

## Когда использовать

- Когда пользователь требует идти через ритуал рабочих продуктов.
- Когда репозиторий уже грязный.
- Когда задача может расползтись на несколько контуров.
- Когда важна отдельная фиксация результата в git.

## Критерий завершения

- slice имеет чёткий `WP`;
- исполнитель и границы объявлены;
- результат материализован в пределах scope;
- git-фиксация сделана отдельно и без доменного загрязнения.
