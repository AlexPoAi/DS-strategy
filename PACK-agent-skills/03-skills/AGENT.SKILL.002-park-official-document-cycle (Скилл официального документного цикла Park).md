# AGENT.SKILL.002 — Скилл официального документного цикла Park

## Назначение

Канонический скилл для повторяемого выполнения официальных документных циклов по проекту Парка:
- письма;
- запросы;
- заявления;
- сохранение clean-docs;
- фиксация в Pack;
- tracking отправки и follow-up.

## Последовательность

1. Определить предмет обращения и правильного адресата.
2. Подтянуть канонические реквизиты отправителя и данные объекта.
3. Довести черновик до clean-version.
4. Сохранить документ в правильное место хранения.
5. Зафиксировать ссылку и статус в Pack.
6. После отправки записать:
   - дату;
   - канал;
   - входящий номер;
   - срок ответа.
7. Поставить follow-up и следующий ход, если ответа нет или он не по существу.

## Проверки после выполнения

- Реквизиты отправителя подтверждены каноническим источником.
- Документ существует не только как черновик, но и как clean-version.
- Место хранения и ссылка зафиксированы.
- Пользователь просмотрел документ и явно согласовал отправку до внешнего send-step.
- Статус документа понятен: `draft / ready-to-send / sent / waiting / answered`.
- Для `waiting` есть follow-up дата.

## Критерий завершения

- Документ доведён до реального рабочего состояния.
- Pack знает, где он лежит и в каком статусе находится.
- Следующий шаг по документу не потеряется без памяти пользователя.
