# AGENT.SKILL.005 — Скилл доведения официальной корреспонденции РФ для Park

## Назначение

Support-skill для `Park Permitting & Infrastructure Coordinator`, который
доводит официальные письма и заявления по Парку до состояния
`print-ready / ready-to-send` по правилам деловой переписки РФ.

Это не юридическое заключение.
Это skill секретарской и документной приёмки.

## Когда запускать

- когда уже понятен адресат и предмет письма;
- когда есть черновик, но его надо довести “под ноль”;
- когда документ пойдёт:
  - в госорган;
  - в муниципальный орган;
  - в ресурсоснабжающую организацию;
  - в официальный внешний контур.

## Что skill обязан проверить

1. `Шапка / адресный блок`
- адресат расположен по правому краю;
- полное или достаточно точное официальное наименование адресата;
- если известно — подразделение / должностное лицо;
- отправитель указан полно и без пустых полей.

2. `Реквизиты отправителя`
- нет заглушек вида `[ФИО]`, `[адрес]`, `[телефон]`, `[email]`;
- реквизиты подтверждены каноническим источником;
- если документ идёт от `ООО ТЕРРА`, это видно прямо и корректно.

3. `Предмет обращения`
- формулировка узкая и проверяемая;
- вопросы нумерованы;
- нет расплывчатой просьбы “разобраться вообще”;
- результат, который мы хотим получить, выражен явно.

4. `Официальный стиль`
- нейтральный деловой тон;
- без разговорных оборотов;
- без эмоциональных формулировок;
- без двусмысленности.

5. `Финальная чистота`
- дата и подпись не забыты;
- блок приложений оформлен;
- документ выглядит как готовый к печати и подаче;
- нет пустых мест, техпометок и следов черновика.

6. `Пагинация / печать`
- если документ по смыслу должен умещаться на 1 лист, агент обязан это проверить;
- подпись, дата и финальный подписной блок не должны уезжать на 2 страницу;
- если после clean-formatting документ всё ещё уходит на второй лист, статус не `print-ready`.

## Жёсткие запреты

- Нельзя оставлять в тексте плейсхолдеры.
- Нельзя считать markdown-черновик финальным письмом.
- Нельзя отдавать пользователю документ как готовый, если шапка не оформлена
  и реквизиты не заполнены.
- Нельзя принимать документ как `print-ready`, если подписной блок съехал на 2 страницу.
- Нельзя маскировать правовую неопределённость красивым оформлением.
- Нельзя подменять этим skill-ом юридическую экспертизу.

## Output contract

После применения этого skill документ должен иметь truthful статус:

- `draft` — ещё есть пробелы;
- `clean-ready` — текст и оформление доведены;
- `print-ready` — можно распечатывать и подавать;
- `ready-to-send` — можно отправлять после user approval.

## Критерий завершения

- документ можно без стыда положить в Google Drive как clean-version;
- пользователь не увидит в нём пустых реквизитов и сырого черновика;
- документ соответствует ожидаемому формату официального обращения в РФ.
