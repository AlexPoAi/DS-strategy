# AGENT.SKILL.006 — Скилл стратега: недельный портфель по контурам

## Назначение

Дать стратегу повторяемый контракт weekly-приоритизации в ситуации,
когда активных контуров много, но фокус недели должен оставаться bounded.

Смысл скилла:

- не свалиться в работу только по одному любимому контуру;
- но и не тянуть все контуры целиком одновременно;
- по каждому живому контуру выбрать один главный ход недели.

## Базовый принцип

`1 контур = 1 главный ход недели`

Это не правило равного времени.
Это правило равного присутствия в weekly-портфеле.

Контур может остаться в активном поле недели, только если для него выбран:

- один главный результат;
- один bounded следующий шаг;
- понятный критерий, зачем он вообще в этой неделе.

## Когда использовать

- На стратегической сессии.
- При сборке или пересборке `WeekPlan`.
- Когда пользователь чувствует расползание в многозадачность.
- Когда контуров много, и есть риск захвата недели одним шумным направлением.
- Когда нужно удержать одновременно:
  - `Парк`
  - `финансы`
  - `Самир`
  - `труба`
  - `менеджерский`
  - `инженерный`
  или другой многоконтурный портфель.

## Последовательность

1. Собрать полный список живых контуров на сегодня.
2. Для каждого контура честно определить статус:
   - `active this week`
   - `support / keep warm`
   - `waiting`
   - `freeze`
3. Для каждого контура со статусом `active this week` выбрать ровно один главный ход недели.
4. Проверить, что главный ход:
   - bounded;
   - не является "весь контур целиком";
   - имеет decision-value или material result уже на этой неделе.
5. Для каждого активного контура зафиксировать:
   - зачем он в weekly-портфеле;
   - что считаем done к концу недели;
   - какой следующий шаг делаем первым.
6. Разложить неделю так, чтобы в день выполнялся один главный ход по одному активному контуру,
   а не происходило хаотичное переключение между всеми сразу.
7. Отдельно отметить, что сознательно:
   - не делаем сейчас;
   - не улучшаем сейчас;
   - не инвестируем сейчас.

## Рабочая матрица

При сборке weekly-портфеля стратег обязан различить:

- `invest` — куда даём внимание, деньги или execution-energy;
- `keep` — что сохраняем в поле, но не разгоняем;
- `freeze` — что временно останавливаем, чтобы не сжигать ресурс.

## Жёсткие запреты

- Нельзя называть weekly-фокусом весь контур целиком.
- Нельзя ставить по одному контуру несколько равноправных главных ходов.
- Нельзя открывать новые улучшения, если не понятны:
  - cost;
  - expected return;
  - влияние на cash discipline.
- Нельзя держать контур в `active this week` только потому, что он тревожит.
- Нельзя подменять weekly-портфель длинным списком задач без выбора.

## Truthful rule

Если контур важен стратегически, но на этой неделе для него нет bounded хода,
стратег обязан честно пометить его как:

- `keep warm`
- `waiting`
- или `freeze`

а не делать вид, что контур реально исполняется.

## Критерий завершения

Скилл выполнен, когда:

- все живые контуры разложены по статусам;
- у каждого `active this week` контура есть ровно один главный ход;
- weekly-портфель не переполнен улучшениями;
- пользователю ясно, что делать по каждому активному контуру дальше;
- зафиксировано, что сознательно не делаем на этой неделе.
