# ENG.CONCEPT.047 — Note-to-WP система (Заметка → РП)

> **Видение:** Экзокортекс как **автоматический конвейер**: заметки → агенты → рабочые продукты → план дня.
>
> **Результат:** Вместо накопления заметок — накопление **готовых РП в inbox/**, которые пользователь выбирает для выполнения.

---

## Поток (как это будет работать)

### День пользователя (текущий день)

```
09:00 — Пишу мысли/проблемы в Obsidian
         ↓ (автоматическая синхронизация)
         fleeting-notes.md в DS-strategy/inbox/
         
       Агенты ночью (cron/scheduler):
         ├─ Note-Review (Strategist) — классифицирует
         └─ Note-to-WP (Strategist) — создаёт РП
         
         WP-NN-*.md появляется в inbox/
         ↓ (регистрируется в 5 местах)
         
08:00 (следующий день) — Day Open
         ↓
       Strategist читает inbox/ как **источник задач**
         ├─ Активные РП (pending, in_progress)
         ├─ По приоритету / проекту
         └─ Показывает в "Требует внимания" → выбери
         
       Пользователь видит:
         🔴 WP-132 Telegram-отчёт (High) — 1.5h
         🟡 WP-133 Google Drive audit (Medium) — 2h
         🟢 WP-134 Тест Recipe (Low) — 0.5h
         
       Выбирает → добавляет в DayPlan → начинает работать
```

---

## Архитектура системы

### 1️⃣ Note-Review (Strategist) — Классификация

**Вход:** `DS-strategy/inbox/fleeting-notes.md` (новые жирные заметки `**текст**`)

**Процесс:**
- Читает заметку
- Классифицирует: НЭП / Задача / Знание / Черновик / Идея / Шум
- Пишет метаинформацию (категория, контекст, параметры)

**Выход:** Классифицированная заметка в памяти (не в файл)

**Статус:** ✅ Уже работает в шаблоне (FMT-exocortex-template)

---

### 2️⃣ Note-to-WP (Strategist) — Создание РП

**Вход:** Классификация "Задача" из Note-Review

**Процесс:**
```
Классификация: "Задача"
    ├─ Текст: "нет health-отчёта в Telegram"
    ├─ Контекст: экзокортекс, видимость
    └─ Параметры (определяет Strategist):
        ├─ name = "Экзокортекс: health-отчёт в Telegram"
        ├─ area = "exocortex-health"
        ├─ repo = "DS-strategy"
        ├─ budget = "1-1.5h" (оценка)
        └─ criteria = "Каждый час в Telegram отчёт о статусе агентов"
    
    ↓
    
Создание РП в 5 местах:
    1. inbox/WP-NN-{slug}.md (основной файл)
    2. WP-REGISTRY.md (реестр)
    3. MEMORY.md (текущая неделя)
    4. WeekPlan таблица (если сегодня вторник-вторник)
    5. WP-context (ссылка на inbox файл)
    
    ↓
    
Удаление заметки из fleeting-notes.md (её жизнь закончена — она стала РП)
```

**Выход:** `inbox/WP-132-exocortex-health-report.md` (готов к выполнению)

**Статус:** ⏳ Нужно создать (ENG.WP.047)

---

### 3️⃣ Day-Open (Strategist) — Чтение РП из inbox/

**Вход:** `DS-strategy/inbox/` (все РП с статусом pending/in_progress/partial)

**Процесс:**
```
Strategist при открытии дня:
    1. Читает ВСЕ РП в inbox/
    2. Фильтрует по статусу (активные)
    3. Сортирует по приоритету/проекту
    4. Показывает пользователю:
        ├─ Carry-over (из вчера)
        ├─ Новые (созданные Note-to-WP)
        └─ Рекомендация по приоритету
    5. Пользователь выбирает → добавляет в DayPlan
```

**Выход:** `DayPlan YYYY-MM-DD.md` с выбранными РП

**Статус:** 🔄 Частично есть (нужно усилить секцию "Требует внимания")

## Как это выглядит для пользователя

### Момент 1: Ночью (автоматический процесс)

```
🕐 23:00 — scheduler запускает Note-Review (headless)

Заметка в fleeting-notes.md:
**Нет health-отчёта в Telegram**
Агенты ломаются и я не узнаю. Нужно добавить hourly отчёт со статусом.

    ↓ Note-Review обрабатывает
    
Классификация: Задача
Параметры: 
  - name: "Health-отчёт агентов в Telegram"
  - budget: "1.5h"
  - area: "exocortex"
  - criteria: "Каждый час / 6 часов отчёт о живых агентах"

    ↓ Note-to-WP создаёт
    
Новый файл:
  inbox/WP-132-exocortex-health-report.md ✅
  
Метаданные:
  - WP-REGISTRY.md: добавлена строка ✅
  - MEMORY.md: "РП текущей недели" + WP-132 ✅
  - WeekPlan: таблица + новая строка ✅
  
Заметка в fleeting-notes.md:
  ~~Нет health-отчёта в Telegram~~ (зачеркнута, как выполненная)

Результат:
  🟢 WP-132 теперь ждёт в inbox/ как готовый к выполнению РП
```

### Момент 2: Утром (Day-Open, 08:00)

```
$ /day-open
↓
Strategist читает inbox/
↓
Показывает в "Требует внимания":

🔴 НОВЫЕ РП (из Note-to-WP ночью):
   • WP-132 Health-отчёт в Telegram (High, 1.5h) — создан вчера 23:15
   • WP-133 Google Drive audit (Medium, 2h) — создан вчера 22:30
   
🟡 CARRY-OVER (из вчера):
   • WP-16 Самокиша вывеска (in_progress) — дедлайн просрочен
   • WP-73 Парк трубе (partial) — дедлайн просрочен
   
РЕКОМЕНДАЦИЯ:
  Сегодня вторник → закрыть вчерашние overdue
  Затем: WP-132 или WP-133 из новых
  
Пользователь видит КОНКРЕТНЫЕ ЗАДАЧИ, не абстрактные заметки.
```

### Момент 3: Выбор для DayPlan

```
Пользователь читает РЕКОМЕНДАЦИЮ и решает:
  → OK, WP-16 первым (overdue)
  → потом WP-132 (интересно, 1.5h укладывается)
  
Strategist добавляет в DayPlan:
| 🔴 | 16  | Самокиша вывеска | 1.5-2h | in_progress |
| 🟡 | 132 | Health-отчёт в Telegram | 1.5h | pending |

Готово → начинает работать
```

## Почему это логично и правильно

### Проблема в текущей системе:
- Заметки копятся как мусор в fleeting-notes.md
- Пользователь должен помнить какие заметки — задачи, какие — идеи
- Нет visibility что ещё осталось (нужно листать файл)
- Note-Review пишет предложения в конец WeekPlan (теряются в деталях)

### Решение Note-to-WP:
- ✅ Заметка → сразу становится РП (виден в inbox/)
- ✅ Accumulation РП, не заметок (это работа, не мысли)
- ✅ Day-Open читает inbox/ как источник задач (ясная визуализация)
- ✅ Пользователь видит приоритеты (есть сортировка, есть фокус)
- ✅ Workflow: мысль → задача → план дня → выполнение (линейный!)

## Технический взгляд

### Входные данные:
1. **fleeting-notes.md** — новые жирные заметки `**текст**`
2. **классификация** из Note-Review (категория, контекст)
3. **параметризация** (budget, area, criteria)

### Выходные данные:
1. **inbox/WP-NN-*.md** — основной РП файл
2. **WP-REGISTRY.md** — новая строка
3. **MEMORY.md** — "РП текущей недели"
4. **WeekPlan** — новая строка в таблице
5. **fleeting-notes.md** — заметка удалена/переархивирована

### Метаданные РП (frontmatter):
```yaml
wp: 132
title: Health-отчёт агентов в Telegram
area: exocortex-health
repo: DS-strategy
budget: 1-1.5h
priority: High
status: pending
criteria:
  - ✅ Каждый час отчёт в Telegram
  - ✅ Видны статусы всех агентов
  - ✅ Ошибки выделены красным
```

| Фаза | РП/Концепция | Время | Цель |
|------|--------------|-------|------|
| 1 | ENG.WP.047 | 4h | Промпт note-to-wp.md создан и протестирован |
| 2 | Update Day-Open | 1-2h | Strategist читает inbox/ как источник задач |
