# ENG.WP.047 — Автоматическая конвертация заметок в РП

> **Проблема:** Note-Review классифицирует заметку как "задача", но новый РП не создаётся автоматически. Требуется ручной вызов `add-wp.md`. Это разрыв в pipeline.
>
> **Цель:** Заметка → классификация → автоматическое создание РП в inbox/ и регистрация в WeekPlan.

---

## Диагностика

### Текущее состояние (как есть)

```
Заметка (fleeting-notes.md)
    ↓
Note-Review (Strategist)
    ├─ Классифицирует: НЭП / Задача / Знание / ...
    └─ Записывает ПРЕДЛОЖЕНИЕ в конец WeekPlan
          ↓
        Пользователь читает предложение
          ↓
        Если согласен → ручной вызов `add-wp.md`
          ↓
        Новый РП создаётся в inbox/
```

### Желаемое состояние (как должно быть)

```
Заметка (fleeting-notes.md)
    ↓
Note-Review (Strategist) — классифицирует: НЭП / Задача / Знание
    │
    ├─ [Знание] → captures.md → Extractor (отдельно)
    │
    ├─ [НЭП] → Dissatisfactions.md (отдельно)
    │
    └─ [Задача] → ЗДЕСЬ РАЗРЫВ
         ↓
       Предложение в WeekPlan (человек видит, одобряет)
         ↓
       Strategist автоматически создаёт WP:
         ├─ Новый файл inbox/WP-NNN-{название}.md ✅
         ├─ Регистрирует в WP-REGISTRY.md ✅
         ├─ Добавляет в WeekPlan таблицу ✅
         └─ Обновляет MEMORY.md ✅
```

---

## Архитектурный разрыв (why это не работает)

### В FMT-exocortex-template есть:

1. **`note-review.md`** (Strategist) — классифицирует заметки, пишет предложения
2. **`add-wp.md`** (Strategist) — ручное добавление РП с параметрами
3. **`inbox-check.md`** (Extractor) — обработка captures.md

### Чего НЕТ:

- **Промпт Note-to-WP** — автоматический переход от классификации "задача" к созданию РП
- **Интеграция между Note-Review и Add-WP** — нет логики "если задача → create WP"
- **Автоматический выпуск WP в 5 мест** — хотя `add-wp.md` может это делать, он не вызывается

### Причина дизайна:

Human-in-the-loop: Note-Review только предлагает, не решает. Это осторожный подход, но он замораживает pipeline.

---

## Решение

### Вариант A: Новый промпт `note-to-wp.md` (рекомендуемый)

**Триггер:** Strategist после Note-Review видит предложение "Задача" → вызывает `note-to-wp.md`

**Алгоритм:**
1. Прочитать предложение из WeekPlan (категория: Задача)
2. Определить параметры:
   - `name` = текст заметки (переформулировать как артефакт)
   - `area` = из контекста заметки
   - `repo` = DS-strategy (по умолчанию, если не указано)
   - `budget` = оценить из контекста (0.5-2h обычно)
   - `criteria` = из описания задачи
3. Вызвать логику `add-wp.md` — создание файла + REGISTRY + MEMORY
4. Удалить предложение из WeekPlan (оно стало РП)

**Статус:** Нужно создать промпт в `/roles/strategist/prompts/note-to-wp.md`

### Вариант B: Интеграция в Note-Review (долгосрочно)

Note-Review сам решает: если задача → сразу создаёт WP. Требует изменения FMT-exocortex-template.
