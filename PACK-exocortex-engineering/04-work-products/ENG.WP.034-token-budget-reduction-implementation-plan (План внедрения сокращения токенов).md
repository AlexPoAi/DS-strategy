# ENG.WP.034 — План внедрения сокращения токенов

**Дата:** 2026-04-11
**Статус:** in_progress
**Основание:** ENG.WP.032 + ENG.WP.033
**Фокус:** безопасное сокращение токенов без поломки локального экзокортекса

---

## Что уже доказано

- upstream Церена даёт реальную экономию токенов;
- у нас можно забрать безопасную Фазу 1 без полного merge;
- основной перерасход сейчас сидит не только в `CLAUDE.md`, но и в слишком раздутом `SESSION-CONTEXT.md`.

## Текущая оценка overhead

| Артефакт | Примерно токенов |
|----------|------------------|
| `CLAUDE.md` | ~2 700 |
| `SESSION-CONTEXT.md` | ~11 000 |
| `MEMORY.md` | ~2 100 |
| `protocol-open.md` | ~1 000 |
| `protocol-close.md` | ~350 |
| `protocol-work.md` | ~600 |
| `USER-PREFERENCES.md` | ~150 |

## Главный вывод

Даже без учёта истории диалога, один вход в сессию может тянуть ~18 000–20 000 токенов системного и контекстного overhead.

Если к этому добавить:
- длинную историю чата,
- tool output,
- повторное чтение больших файлов,
- многоходовые инженерные сессии,

то расход в `~1 000 000` токенов за несколько интенсивных запросов уже не выглядит невозможным.

## Источники перерасхода в порядке приоритета

### 1. `SESSION-CONTEXT.md`

Самый тяжёлый артефакт.
Проблема:
- файл слишком длинный;
- в нём исторический хвост за много дней;
- он используется как живой handoff, но фактически стал логом.

### 2. Длинная история самой сессии

Когда сессия не обрезается и внутри много рабочих циклов, overhead начинает расти быстрее самих задач.

### 3. `CLAUDE.md`

Он не критически огромный, но всё ещё содержит много правил, которые можно вынести в `.claude/rules/`.

### 4. Повторная загрузка протоколов и memory-слоя

Это нужно, но можно сделать lazy / thinner.

## План внедрения

### Фаза 0 — быстрый выигрыш без риска

1. Сжать `SESSION-CONTEXT.md` до короткого handoff:
   - статус;
   - 5–10 последних значимых шагов;
   - текущий активный WP;
   - next step.
2. Старый хвост вынести в архивный лог, а не держать в живом контексте.
3. Зафиксировать правило:
   - длинные инженерные работы вести через отдельные WP и чаще закрывать/обнулять контекст.

### Фаза 1 — безопасное заимствование из upstream

1. Внедрить `.claude/rules/`
2. Вынести formatting/distinctions из `CLAUDE.md`
3. Добавить `extensions/`
4. Добавить `params.yaml`
5. Адаптировать безопасные skills

### Фаза 2 — осторожная модернизация

1. Сжать `CLAUDE.md`
2. Адаптировать `day-open` skill под нашу экосистему
3. Проверить hooks и gates после изменений

## Практическая рекомендация

Если цель — быстро остановить перерасход токенов, то первым делом надо трогать не upstream merge, а:

1. `SESSION-CONTEXT.md`
2. дисциплину коротких сессий / закрытия WP
3. только потом `CLAUDE.md` и `.claude/rules/`
