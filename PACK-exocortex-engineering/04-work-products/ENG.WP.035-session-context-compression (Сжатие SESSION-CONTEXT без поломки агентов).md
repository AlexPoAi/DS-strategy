# ENG.WP.035 — Сжатие SESSION-CONTEXT без поломки агентов

**Дата:** 2026-04-11
**Статус:** completed
**Основание:** ENG.WP.034 + команда `сверь с эталоном`

---

## Эталон

По эталонной модели экзокортекса:
- `SESSION-CONTEXT.md` — это живой handoff;
- он нужен для входа в следующую сессию;
- он не должен быть историческим логом всех событий;
- агентам важны путь файла, свежесть, timestamp и базовые секции, а не длинный архивный хвост.

Источники эталона:
- `exocortex/project-brain.md`
- `exocortex/architecture.md`
- `ENG.WP.031`

## Текущее состояние

- `SESSION-CONTEXT.md` разросся до ~11k токенов;
- в него смешаны:
  - живой handoff,
  - исторический лог,
  - следы старых инженерных циклов;
- это увеличивает стоимость каждой сессии и мешает truthful compact open-flow.

## Разрыв

Эталон:
- короткий, рабочий, актуальный handoff.

Сейчас:
- полу-журнал, полу-handoff.

Главный разрыв:
- файл выполняет сразу две роли, которые должны быть разведены.

## Решение

Разделить контур на два слоя:

1. `current/SESSION-CONTEXT.md`
   - только живой handoff
2. архивный лог
   - отдельный файл / папка в `archive/`

## Что нельзя ломать

- путь `DS-strategy/current/SESSION-CONTEXT.md`
- timestamp-обновление в `close-task.sh`
- секции, которые переписывает `close-task.sh`
- freshness check в scheduler/health-check

## Что можно менять безопасно

- длину файла
- объём блока `Что сделано сегодня`
- исторический хвост
- структуру архивирования старых записей

## Следующий шаг

Собрать новый компактный шаблон `SESSION-CONTEXT.md` и отдельный архивный файл для старого хвоста, затем проверить совместимость с `close-task.sh`.
