# ENG.WP.046 — Жёсткий gate закрытия инженерной итерации

## Инцидент

Во время работы по складскому контуру инженер допускал разрыв между:

- выполнением локальной правки,
- ручной проверкой,
- и фактическим закрытием итерации через `commit/push/clean-tree`.

Из-за этого в дереве оставались незавершённые хвосты и generated-артефакты,
а факт «итерация завершена» больше не совпадал с реальным состоянием репозитория.

## Root cause

В skill инженера не было явно materialized exit gate:

- нельзя продолжать работу без `git status`;
- нельзя считать шаг завершённым без `push`;
- нельзя писать `готово`, если собственный хвост остался в дереве.

## Принятое решение

Ввести отдельный `Iteration Closure Gate` как обязательную часть инженерного ритуала.

## Closure Gate

Инженер не имеет права переходить к следующему инкременту, пока текущий не закрыт циклом:

1. `manual check`
2. `git status`
3. `commit`
4. `push`
5. повторный `git status`

## Truthful language rule

Инженеру запрещено использовать слова:

- `сделано`
- `закрыто`
- `готово`

если:

- нет `push`,
- нет повторной проверки `git status`,
- или в дереве остаётся его собственный незавершённый хвост.

## Generated artifacts rule

Generated-файлы не считаются «мелочью».

Они должны:

- либо входить в commit осознанно как часть результата,
- либо очищаться сразу,
- но не переноситься в следующий рабочий ход по инерции.

## Итог
